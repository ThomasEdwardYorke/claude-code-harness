# 言語規約

## コミットメッセージ形式

Conventional Commits に基づき、以下の形式を使用する:

```
<prefix>: <サマリ（日本語・50文字以内）>

- 変更内容1
- 変更内容2
```

| Prefix | 用途 |
|--------|------|
| `feat` | 新機能の追加 |
| `fix` | バグ修正 |
| `refactor` | リファクタリング（挙動変更なし） |
| `test` | テスト追加・修正 |
| `docs` | ドキュメント更新 |
| `chore` | 雑務（ツール設定・スクリプト等） |
| `style` | スタイルのみの変更 |
| `revert` | 取り消し |
